\section{Experimental Evaluation}
\label{sec:exp}

\subsection{Experimental Setup}

We evaluate all four algorithms on mathematical reasoning tasks under two settings:

\textbf{Small-scale comparison:} Qwen2.5-0.5B-Instruct as the base model, 10 sample problems from GSM8K/MATH, single NVIDIA GPU training for up to 500 steps. Each algorithm uses its default hyperparameters.

\textbf{Large-scale benchmark:} Qwen2.5-32B as the base model, evaluated on AIME 2024 (30 challenging competition problems), using metrics avg@$k$, pass@$k$, and cons@$k$ (consensus@$k$) with $k = 32$ samples per problem.

All algorithms converge within 400 steps, but DAPO shows the steepest descent in the early phase (steps 0--200) due to dynamic sampling ensuring informative gradients.

\begin{enumerate}[leftmargin=*, itemsep=1pt]

\item \textbf{Clip-Higher} adds +2 points by maintaining exploration throughout training.
\item \textbf{Token-Level Loss} adds +1 point by removing the length bias, though its effect is relatively modest in isolation.
\end{enumerate}

\section{Discussion and Future Directions}
\label{sec:discuss}

\subsection{Practical Algorithm Selection Guidelines}

Based on our analysis, we provide the following guidelines:

\begin{itemize}[leftmargin=*, itemsep=2pt]
\item \textbf{Choose DPO} when: high-quality preference data is available; the task involves subjective alignment (helpfulness, safety); simplicity of implementation is prioritized; GPU resources are limited.
\item \textbf{Choose GRPO} when: the task has objective, rule-based rewards; memory efficiency is critical; the model size is small enough that entropy collapse is less severe; rapid prototyping is needed.
\item \textbf{Choose DAPO} when: maximum reasoning performance is required; long chain-of-thought reasoning is involved; sufficient compute for group sampling is available; training stability over long runs is important.
\item \textbf{Choose PPO} when: a learned reward model is required; the task involves general RL beyond LLM alignment; theoretical convergence guarantees are needed; the infrastructure already supports PPO-based RLHF.
\end{itemize}

\subsection{Open Challenges}

\begin{enumerate}[leftmargin=*, itemsep=2pt]
\item \textbf{Scalability to 100B+ Parameters:} All four algorithms face significant engineering challenges at frontier model scales. Group-based methods (GRPO, DAPO) require $G$ forward passes per prompt, which becomes prohibitively expensive for very large models.
\item \textbf{Reward Specification for Open-Ended Tasks:} While mathematical reasoning benefits from rule-based rewards, extending these methods to creative writing, dialogue, and open-ended generation remains challenging.
\item \textbf{Length Exploitation:} Models may learn to game length-based reward signals (e.g., generating unnecessarily long chain-of-thought reasoning). DAPO's overlong reward shaping partially addresses this, but more robust solutions are needed.
\item \textbf{Sample Efficiency:} Group methods require $G$ samples per prompt per training step, which is $G$ times more expensive than single-sample methods. Reducing this overhead while maintaining advantage quality is an important open problem.
\item \textbf{Multi-Objective Optimization:} Real-world LLM deployment requires balancing accuracy, safety, helpfulness, and efficiency simultaneously. Current algorithms optimize a single scalar reward, which may not capture these trade-offs optimally.
\end{enumerate}

\subsection{Promising Research Directions}

\begin{enumerate}[leftmargin=*, itemsep=2pt]
\item \textbf{Hybrid Approaches:} Combining DPO's offline preference learning with DAPO's token-level online optimization could leverage the strengths of both paradigms.
\item \textbf{Process Reward Models (PRM):} Step-level reward signals~\cite{lightman2024verify, uesato2022solving, wang2024mathshepherd} could replace outcome-level rewards, providing richer training signals for chain-of-thought reasoning.
\item \textbf{Adaptive Group Sampling:} Dynamically adjusting $G$ based on prompt difficulty could reduce computational waste while maintaining advantage quality.
\item \textbf{Speculative Decoding for Group Sampling:} Using speculative decoding to generate group samples more efficiently could significantly reduce the overhead of GRPO/DAPO.
\item \textbf{Meta-Learning for Algorithm Selection:} Training a meta-learner to select the optimal RL algorithm and hyperparameters for a given task could automate the algorithm selection process.
\end{enumerate}

\section{Conclusion}
\label{sec:conclusion}

This survey has provided a comprehensive and systematic analysis of four reinforcement learning algorithms for large language model training: PPO, DPO, GRPO, and DAPO. Our contributions include a unified mathematical framework with consistent notation, detailed architectural comparisons, and empirical evaluation on mathematical reasoning benchmarks.

Our analysis reveals a clear evolutionary trajectory in the field:
\begin{itemize}[leftmargin=*, itemsep=1pt]
\item \textbf{PPO} (2017) established the clipped surrogate foundation and dual-network architecture that enabled the first generation of aligned LLMs (InstructGPT, ChatGPT).
\item \textbf{DPO} (2023) demonstrated that the RLHF pipeline could be dramatically simplified by eliminating explicit reward modeling, trading online exploration for offline simplicity.
\item \textbf{GRPO} (2025) eliminated the value network through group advantage normalization, reducing memory by $\sim$50\% and simplifying the architecture for verifiable reasoning tasks.
\item \textbf{DAPO} (2025) extended GRPO with three targeted innovations---asymmetric Clip-Higher, dynamic sampling, and token-level loss---achieving a 67\% relative improvement on AIME 2024.
\end{itemize}

For practitioners, our recommendation is clear: use \textbf{DPO} for subjective alignment tasks with available preference data, \textbf{GRPO} for efficient reasoning with limited resources, \textbf{DAPO} for maximum reasoning performance, and \textbf{PPO} when a learned reward model and theoretical guarantees are required. The field continues to evolve rapidly, with hybrid approaches and process-level feedback representing particularly promising directions for future research.
